# Docs

We keep all the docs in the .agent folder and keep updating them, structure as follows:

.agent
- Tasks : PRD & implementation pla for each features
- System : Document of the current system (project structure, tech stack, integration points, database schema, and core functionalities, etc.)
- SOP: Best practices of executing certain tasks (e.g. code review process, git workflow, etc.)
- Readme.md : an index of all the docs

We should always update the .agent folder when we make any changes to the system to make sure it reflects the up to date information.

Before you plan any implementation, always read the .agent/Readme.md first to get context
